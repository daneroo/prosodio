# corpora

What must hold about the audiobook/ebook corpora — where metadata truth lives
and how the corpora are validated. The corpora themselves: `fixtures/` (public,
committed), the private library, and the `nx-audiobook` staging area (see
[validation.md](validation.md) for the full story).

- [metadata.md](metadata.md) — the truth hierarchy (m4b tags canonical, basename
  a flagged fallback), the `grouping` series grammar
- [validation.md](validation.md) — corpus validation charter: one core, CLI +
  web skins, three corpora; milestones to nx-audiobook parity
